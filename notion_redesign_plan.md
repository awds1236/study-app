# Notion 스타일 리디자인 기획서

## 목적

현재 공부 보조 AI의 기능 흐름은 유지하고, 화면 구조와 시각 언어만 Notion식 워크스페이스 UI로 재정리한다. 함수, 이벤트 핸들러, API 호출, 상태 관리, `id/name/data-*` 속성은 유지한다.

## Notion 관찰 요약

- Notion 홈페이지는 강한 첫 인상 구간에서 deep navy 배경, 큰 헤딩, 보라색 CTA, 보조 CTA를 조합한다.
- Notion의 제품/가격 페이지는 흰 캔버스, 1px hairline border, 12px 카드, 8px 버튼, 촘촘하지만 넉넉한 정보 그리드를 사용한다.
- 보라색은 핵심 CTA에만 쓰고, 나머지 버튼은 검정/아웃라인/고스트 위계로 정리한다.
- 파스텔 카드(peach, mint, sky, lavender, yellow)는 정보 단위를 구분하는 데 쓰되, 전체 배경을 강하게 물들이지는 않는다.

## 적용 방향

1. 앱 전체는 Notion 워크스페이스처럼 `surface-soft` 위의 흰 캔버스 카드 구조로 만든다.
2. 사이드바는 `canvas` 배경, `hairline` 보더, 검정 활성 메뉴, 2열 모드 스위처로 구성한다.
3. 대시보드는 큰 마케팅 hero가 아니라 업무 도구용 문서 헤더로 낮춘다.
4. 통계 카드 4개는 `card-tint-peach`, `card-tint-mint`, `card-tint-sky`, `card-tint-lavender` 순서로 유지한다.
5. 빠른 시작은 `card-tint-yellow-bold` callout으로 유지하되, 첫 버튼만 보라색 CTA로 둔다.
6. 나머지 페이지(업로드, 개념 정리, 문제 풀기, 설정)는 동일한 카드/입력/탭/퀴즈 컴포넌트 시스템을 공유한다.
7. 모바일에서는 상단 모드바와 하단 내비가 튀어나오지 않도록 가로 스크롤 또는 축약된 터치 타깃으로 정리한다.

## 구현 범위

- `index.html`의 CSS만 수정한다.
- 필요 시 클래스명은 스타일 목적에 한해 조정할 수 있으나, 이번 적용은 CSS override 중심으로 진행한다.
- 동작 연결점인 `onclick`, `id`, `name`, `data-*`, 파일 입력, 모달, Firebase/Gemini 관련 로직은 수정하지 않는다.

## 성공 기준

- 사이드바 모드 버튼이 본문 영역으로 튀어나오지 않는다.
- 대시보드 제목이 화면을 압도하지 않고, 앱 대시보드 헤더 크기로 보인다.
- 본문 배경은 어색한 navy/white 단절 없이 `surface-soft` 기반으로 정돈된다.
- 통계/빠른 시작/학습 현황/최근 기록 카드가 같은 디자인 시스템 안에 놓인다.
- 기존 기능 플로우는 그대로 유지된다.
